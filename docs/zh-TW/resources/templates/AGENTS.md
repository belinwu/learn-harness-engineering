# AGENTS.md

這個儲存庫面向長時執行的 coding agent 工作流。目標是讓每一輪工作階段結束後，下一個工作階段仍然能無猜測地繼續工作，而不是儘可能快地產出程式碼。

## 開工流程

寫程式碼前先做這些事：

1. 用 `pwd` 確認目前目錄。
2. 讀取 `claude-progress.md`，瞭解最新已驗證狀態和下一步。
3. 讀取 `feature_list.json`，選擇優先級最高的未完成功能。
4. 用 `git log --oneline -5` 看最近提交。
5. 執行 `./init.sh`。
6. 在開始新功能前，先跑必需的 smoke test 或端到端驗證。

如果基礎驗證一開始就失敗，先修基礎狀態，不要在壞的起點上繼續疊新功能。

## 工作規則

- 一次只做一個功能。
- 不要因為“程式碼已經寫了”就把功能標記為完成。
- 除非為了消除目前 blocker 的窄範圍修復，否則不要擴大到其他功能。
- 實現過程中不要悄悄改弱驗證規則。
- 優先依賴儲存庫裡的持久化檔案，而不是聊天記錄。

## 必需檔案

- `feature_list.json`：功能狀態的唯一事實來源
- `claude-progress.md`：工作階段進度和目前已驗證狀態
- `init.sh`：統一的啟動與驗證入口
- `session-handoff.md`：較長工作階段可選的交接摘要

## 完成定義

一個功能只有在以下條件都滿足時才算完成：

- 目標行為已經實現
- 要求的驗證真的跑過
- 證據記錄在 `feature_list.json` 或 `claude-progress.md`
- 儲存庫仍然能按標準啟動路徑重新開始工作

## 收尾

結束工作階段前：

1. 更新 `claude-progress.md`
2. 更新 `feature_list.json`
3. 記錄仍未解決的風險或 blocker
4. 在工作處於安全狀態後，用明確的提交資訊提交
5. 保證下一輪工作階段可以直接執行 `./init.sh`
