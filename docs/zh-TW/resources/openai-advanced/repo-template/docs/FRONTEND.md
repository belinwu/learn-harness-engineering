# FRONTEND.md

這份檔案定義穩定的前端預期，避免 agent 每次都臨場發明一套 UI 模式。

## UI 原則

- 先保證易懂，再追求新鮮感。
- 互動流程要可發現、可重走、可重啟。
- 優先沉澱少量可複用元件，而不是到處長一次性變體。
- 無障礙檢查屬於正常驗證，不是最後的美化工作。

## 護欄

- 把設計系統或元件庫參考材料放進 `docs/references/`。
- 顯式記錄關鍵使用者狀態：empty、loading、success、error、retry。
- 在不同流程裡保持文案、鍵盤行為、視覺層級一致。
- 修復 UI bug 後，順手補上對應驗證步驟。

## 驗證要求

- 為關鍵使用者旅程留下證據。
- 把瀏覽器或執行時期驗證步驟寫進相關 plan。
- 如果視覺迴歸常見，就標準化截圖或 DOM 檢查。
